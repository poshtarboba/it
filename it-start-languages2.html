Склад мови програмування

Ядро мови програмування — це її математично-логічний блок.

Кожна мова програмування повинна вміти додавати, віднімати, множити, ділити, виконувати інші математичні операції. Працювати не тільки з байтами, але і бітами. Вміти порівнювати величини і значення. Обробляти помилки.

В залежності від призначення далі на мову "навішуються" додаткові модулі, як навісне обладнання на трактор:

Наприклад, якщо взяти мову JavaScript - в браузері на ядро навішано модулі для роботи з веб-сторінкою і передачі даних через інтернет. Якщо JavaScript встановити на сервер - додаються модулі для роботи з файлами і деякі інші. Якщо JavaScript запхати в іграшку-робота, то мові не потрібні модулі для роботи з файлами та веб-сторінками, а ось для передачі інформації по інтернету - може знадобитися. А також спеціальний модуль, який дозволить керувати руками, ногами, головою, динаміком з мікрофоном та камерою, різними лампочками.

Бібліотеки (функцій)

Писати на голій мові програмування з нуля - досить довго і складно.

Окрім того, скоріш за все хтось раніше вже писав ці самі чи подібні програми, реалізовував алгоритми. Заново з нуля те саме писати не продуктивно.

Тому часто використовувані шматки коду збирають у файли і називають їх бібліотеками функцій.

Такий приклад: в коді нам часто потрібно перетворювати ПІП людини на її прізвище та ініціали:

Іванов Петро Григорович ==> Іванов П.Г.

Абстрактний код буде приблизно такий (на відміну від шкільної математики в програмуванні символ дорівнює записується зліва, а дії над значеннями виконуються справа, ми мов би кажемо: "персона1 - це строка тексту; піб - це розбита на слова строка тексту"):

персона1 = "Іванов Петро Григорович";
персона2 = "Сидорчук Андрій Васильович";
персона3 = "Мельник Лариса Теодорівна";
// розбиваємо ПІП на окремы слова по пробілу
// збираємо нову строку тексту з прізвищем та ініціалами
ПІП = розбитиНаСлова персона1;
персона1 = ПІП(1) + " " + ПІП(2)буква(1) + "." + ПІП(3)буква(1) + ".";
ПІП = розбитиНаСлова персона2;
персона2 = ПІП(1) + " " + ПІП(2)буква(1) + "." + ПІП(3)буква(1) + ".";
ПІП = розбитиНаСлова персона3;
персона3 = ПІП(1) + " " + ПІП(2)буква(1) + "." + ПІП(3)буква(1) + ".";

Такий код важкувато читати, і тричі виконується одне і те саме. В реальних програмах шматки коду можуть виконуватися тисячі і мільйони разів.

Винесемо код, що створює нову строку з ініціалами в коремий шматок коду:

шматокКоду прізвищеІніціали (
  ПІП = розбитиНаСлова персона;
  персона = ПІП(1) + " " + ПІП(2)буква(1) + "." + ПІП(3)буква(1) + ".";
);
персона1 = прізвищеІніціали "Іванов Петро Григорович";
персона2 = прізвищеІніціали "Сидорчук Андрій Васильович";
персона3 = прізвищеІніціали "Мельник Лариса Теодорівна";

Бібліотеки функцій зазвичай містять в собі десятки, сотні, тисячі функцій, які значно пришвидшують написання коду, роблять його компактнішим та таким, що краще читається.

Фреймворки

Окрім шматків коду (функцій) часто використовуються одні й ті ж елементи інтерфейсу (кнопки, панелі, графіки, повзунки, таблиці, списки, поле пошуку, форми логінації і т.п.).

Щоразу оформляти кожну кнопку чи повзунок, а також програмувати її поведінку - теж задача не з легких, але рутинних. Тому такі елементи також збирають у "бібліотеки", які називають фреймворками. І з фреймворків досить легко брати той чи інший елемент і використовувати в своїх програмах.

Зустрічається також назва "Бібліотека компонентів".

Таким чином, фреймворк - це набір шаблонних рішень, заготовок для інтерфейсу, алгоритмів, що значно пришвидшують розробку проектів і в деякій мірі збільшують їх надійність (оскільки зменшується кількість потенційних помилок - використовуються перевірені часом і багатьма розробниками рішення).

Популярний фреймворк для розробників сайтів (клієнтської частини) - Bootstrap, фреймворки для серверної мови PHP - Laravel, Symphony, Zend, фреймворки для Python - Django, Flask...
